# PPS — Patrol Priority Scorer 운영 스킬

PPS는 규칙+통계 기반 스코어링 모델이다. 과거 패트롤 데이터에서 통계를 뽑아
(`knowledge_base.json`) 새 데이터에 등급·코드·점수·사유를 부여한다.
신경망/ML 프레임워크는 쓰지 않으며 모든 점수는 설명 가능하다.

작업 디렉터리: 이 스킬이 있는 프로젝트 루트. 파이썬은 프로젝트의 표준 인터프리터 사용.

## 언제 어떤 작업을 하나

### 1) 예측 — "이 데이터 우선순위 매겨줘" / "예측해줘"
새 패트롤 엑셀(.xlsx)에 등급을 부여한다.
```
python predict_new.py <엑셀파일> [출력접두어]
```
→ `<접두어>_예측결과.csv` (MES 인입용, 원본순서) + `<접두어>_예측결과.xlsx` (색상)
- 출력 컬럼: 공정군 · 등급(긴급/주의/일반) · 등급코드(1/2/3) · 점수 · 사유
- 몇 건만 채팅으로 받으면 `patrol_api.score_records([{...}])`로 직접 채점해 표로 보여준다.

### 2) 재학습 — "재학습해줘" / "이번 달 데이터 반영해줘"
누적 전체 데이터로 모델을 갱신한다.
```
python train_model.py <누적_전체데이터.xlsx> [추가파일...]
```
→ `knowledge_base.json` 갱신. 여러 달 파일을 함께 줄 수 있다(중복 자동 제거).
운영 순서: 월말에 그달 데이터를 합쳐 재학습 → 다음 달 예측은 더 정밀.

### 3) 설명 — "이 건 왜 긴급이야?" / "등급 기준 알려줘"
- 각 예측의 `사유` 컬럼이 이미 산정 근거를 담는다(예: "심함 · 단기 빈발 · 다발설비").
- 등급 기준 전체는 `등급기준.md` 참조. 핵심:
  - 열처리·기타: 점수컷(긴급≥60 / 주의≥25 / 일반<25)
  - 스프링: 와이어 이탈=긴급, 와이어 튐=주의, 다이스 마모=일반(유형 고정), 그 외 점수컷
  - 점수 = 기본10 × 유형위험도 × 심함(2.2) × 단기빈발(1.35~2.1) × 다발설비(1.3) × 조치요청(1.25)

### 4) 대시보드 — "화면으로 보고 싶어"
```
streamlit run dashboard.py
```
→ 브라우저에서 엑셀 업로드, 등급 분포·우선 리스트·필터·다운로드, 등급컷 실시간 조정.

## 입력 규격
필수: `불량처리 일시` · `공정` · `설비` · `불량내용`
선택: `설비불량유형`(드롭다운, 텍스트 오타 시 안전망) · `검사원` · `양호조치일시` · `비고`

## 엔진 파일 (수정 시 주의)
- `patrol_lib.py` — 표준화+채점 핵심. 상단 상수로 조정: `SPRING_TYPE_GRADE`(스프링 유형→등급),
  `grade_for(urgent=60,warn=25)`(등급컷), `TYPO_NORMALIZE`(오타사전), `DROPDOWN_MAP`(드롭다운 매핑),
  `burst_half_life_days`(단기빈발 반감기).
- `patrol_api.py` — `score_records(records)` 호출 인터페이스(DB 연동 접점).
- 가중치/기준을 바꾼 뒤에는 반드시 `train_model.py`로 재학습해야 반영된다.

## 주의
- 실제 공장 데이터·검사원 실명은 외부(공개 저장소 등)로 내보내지 않는다.
- 예측은 "보고된 불량의 우선순위"를 매기는 것이지, "불량 발생 여부"를 예측하는 것이 아니다.
